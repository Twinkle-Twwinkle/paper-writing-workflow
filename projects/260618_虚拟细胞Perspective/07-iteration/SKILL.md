# 论文迭代（= 带着新 context 重走同一条 00→06 管线）

> **一句话定位**：`00→06` 主流程解决「从无到有写出初稿」；迭代解决「从有到更好」。但**迭代不是另一套系统**——它就是同一条管线、换一轮输入再跑一遍。
>
> **设计目标**：清爽、可审计、人在环。**保持简单**——能跳过的步骤就跳过，不该加的基础设施一律不加。

---

## 心智模型（务必先认这个）

```
新 context → 00-context/  →  沿 01→02→03→04→05→06 同一条管线推进
                              （没变的步骤核验后跳过，但顺序不许乱）
                              →  06-outline-and-draft/vN/{outline,draft}.md（旧版只读）
                              →  07-iteration/迭代日志.md 记一条 + CURRENT → vN
```

同一套框架反复支撑不同轮次。导师/reviewer 意见、新范围界定、新文献——**全都先变成 `00-context/` 里的一个新文件**，然后照常往下走。

---

## 开工前先定位（必做）

1. **读 `07-iteration/迭代日志.md` 顶部的 `CURRENT: vN`** —— 这是本轮基线。绝不靠记忆或时间戳猜哪个最新。
2. 取基线版 `06-outline-and-draft/vN/{outline,draft}.md`（首版无此项，直接据 `00-context/` 起）。
3. 确认本轮**新 context** 已落成 `00-context/` 里的一个新文件（导师/reviewer 意见、新范围、新文献要点、自查发现）。**没有就先索要 / 帮用户整理。**
4. **若用户只给「新意见」没指明基线版本，先问清，不要假设。**

---

## 一轮迭代 = 五步

### Step 1 · 新 context 进 00（先保真，后提炼）
- 把新输入**逐字原话**存档到 `00-context/` 的一个新文件，标注**来源、日期、类型**；**不改写原话、不覆盖旧 context**。
- 另起一段**提炼要点**，与原话区物理分开。范本：`00-context/郭天南新增意见汇总_260622.md`。
- **若新意见改变了"本文要找哪些证据"**：由**人**更新 `00-context/evidence-questions.md`（固定 ClaimID 清单）。只是写作口径 / 措辞 / 范围边界 → 不一定需要新 ClaimID。

### Step 2 · 判断本轮重跑范围（能跳就跳）
对照新 context，判断 `01→05` 要不要重跑：
- **只改写作口径 / 范围 / 措辞，无新增文献** → `01–05` 核验后**全跳过**，直奔 06。
- **有新增文献** → 从 `01` 一路补到 `05`（下载→清洗→精练）。**调用** `论文下载`/`论文清洗`/`精练` skill，本 skill 不自己做这些。
- **文献主库只增不改**：新文献只追加，不复制整套库；**已蒸馏的论文默认不重做**，只有新 ClaimID 明显需要旧论文里没抓出的证据时，才挑少数旧论文做 **targeted backfill**。
- 新蒸馏 / backfill 的卡，`claimId` 一律直接用 `evidence-questions.md` 的规范 key。

> ⚠️ **不每轮扇出重读全部文献。** 上一版已审过的证据沿用 `05-distilled/cards.json`；只读本轮真正动到的那几篇。

### Step 3 · 出新版 outline（先动骨架）
在 `06-outline-and-draft/vN/outline.md` 上，以基线版为底**最小改动**：
- 只改新 context 触及的节，其余原样保留；不借机重写无关章节。
- 每条新增 / 改动论断仍绑 `[引用号] + 逐字原句`（取自 `cards.json`）。
- **显式标注本轮改动**（如 `【v2 新增】`），方便审计回溯。
- 证据撑不住的新论断 → 退回 Step 2 补语料，或降级 / 删除，**不许硬写**。

### Step 4 · 重编 draft（先骨架后成文）
由改后的 outline **重新编译** `06-outline-and-draft/vN/draft.md`：
- 守住期刊约束（字数上限、参考文献条数、article type）。
- 不留半句无引文支撑的话；**意思的改动只能来自 outline，不在 draft 上直接改意思。**
- **旧版 v{N-1} 只读保留，不覆盖。**

### Step 5 · 审计 + 登账（防改崩 + 留痕）
- **逐字回搜审计**：抽查每条新增 / 改动引用能否在 `04-markdown-clean/*` 原文里逐字命中，零编造、零改写；顺带确认旧版已审论断没被改坏。
- **登账**：往 `07-iteration/迭代日志.md` 顶部追加一条 changelog（模板见文末），把 `CURRENT` 指到本版。审计不通过则不进版本号，回对应 Step 修。

---

## 版本管理

```
06-outline-and-draft/
  v1/ outline.md  draft.md      # 只读快照
  v2/ outline.md  draft.md      # 本轮产出
07-iteration/
  迭代日志.md                    # 顶部 CURRENT: vN + 每轮一条 changelog
```
- 主版本 `vN`；小修可 `vN.1`；废弃 / 走偏的尝试进 `_archive/` 留痕。
- 旧版本一旦定版即不可变。**当前接受版本只认 `迭代日志.md` 顶部的 `CURRENT`。**

---

## 硬约束（全程遵守）

1. **迭代 = 重走同一条管线**，不是另一套系统。
2. **基线只认 `CURRENT` 指针**；收尾把它指到本版。
3. **文献主库唯一、只增不改**：01–05 永远只维护根目录那一份，不分版、不复制。已蒸馏不重做，只 targeted backfill。
4. **ClaimID 由人维护**：固化在 `00-context/evidence-questions.md`，**agent 不许自造编号 / 名称**。
5. **主产物 vs 检查视图分离**：下一步真正依赖的 canonical artifact（如 `cards.json`）放显眼处；给人核验的派生视图（evidence-matrix / citation-registry）放 `_inspection/`，可删可重建，不是 source of truth。
6. **旧版只读、每轮登账。**

## 明确不做（保持半自动简单稳定，导师定）

- ❌ 不建 `06-iterations/` 去替代 `06-outline-and-draft/`。
- ❌ 不每轮强制扇出重读全部文献。
- ❌ 不上 CSV 台账 / 一致性脚本（除非项目后续确实需要）。
- ❌ 不建每轮 `runs/<轮次>/` 过程目录；材料一律回写 `00–06` 对应目录。
- ❌ 不做自动 dispatcher / `STATE.json` / 一键全流程。真正要维护的是每步的「输入/输出/核验合同」。
- ❌ 不从零写初稿（那是 00→06 主流程）；不替用户判断意见对错（忠实纳入 + 提醒证据缺口）；不擅自改地基 context（只增量追加）。

---

## 引用纪律

- **可正式引用**：同行评审期刊、正式综述 / Perspective、正式会议论文。
- **不可正式引用**：预印本（未见刊）、editorial / news-feature、博客、白皮书 → 标 `[idea, not cited]`，尽量给已发表替代，绝不进参考文献。
- **老师 / 评审 / webinar / 私下讨论**：只作 motivation 语气，注明出处，写进 `00-context/` 新文件留痕，**永不进参考文献**。
- **不臆造文献**：本版每篇引用都必须在文献主库真实存在；发表状态拿不准 → 标 `meta?=⚠`，不假定可引。
- **引用挂稳定 key**：正文用稳定引用号 / `[Author Year]`，**不用会逐版重排的 `[1][2]`**；统一数字编号只在最终导出参考文献时做一次。

## 迭代日志 changelog 模板（`07-iteration/迭代日志.md` 每轮追加一条）

```markdown
## v{N} · {日期} · from v{N-1}
- **输入 context**：{类型：PI意见/reviewer/新文献/自查} —— {一句话 + 原话存档文件链接}
- **本轮重跑**：{如 00, 05, 06；跳过了哪些}
- **是否补语料**：{否 / 是：补了哪几篇，新卡落在 05-distilled 何处}
- **outline 改动**：{标了【v{N} 新增/修改】的条目}
- **draft 改动**：{受影响章节 + 字数变化}
- **审计结论**：{PASS / 不通过原因} —— 逐字回搜 {抽查了哪几条引用}
- **产物**：`06-outline-and-draft/v{N}/{outline,draft}.md`
```
