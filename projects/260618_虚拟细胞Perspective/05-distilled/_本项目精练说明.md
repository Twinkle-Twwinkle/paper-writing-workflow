# 本项目 05-distilled 说明

本项目的精练**不是**仓库默认的「精练3.0 → 每篇一个 `<basename>_refined.md`」markdown 形态，而是升级到 **精练4.0（直接吐 JSON）** 并做了下游聚合，因此本目录的产物是：

| 文件 | 是什么 | 来自 |
|---|---|---|
| `cards.json` | 41 篇论文的 **excerpt-centric JSON 证据卡**合集（每条逐字原句自带 `touchpoints:[{claimId,direction}]`）——下游唯一数据源 | P1 抽取（精练4.0） |
| `样例卡-08a_refined.json` | 单篇卡样例（看结构用） | P1 抽取 |
| `evidence-matrix.md` | 把 41 张卡按 claim **转置**成的证据矩阵 + scoreboard + 逐字引文 + 涌现主题 | P2（纯代码） |
| `citation-registry.md` | 引用注册表（参考文献种子）+ **审计红线**：`meta?=⚠` 标的 33 篇期刊/DOI 是推断的，编译期须核验 | P2（纯代码） |

> 为什么用 4.0/JSON：3.0 产 markdown 后下游还要把 md 解析回 JSON，是冗余且有损的往返（每 claim 只留 1 句，丢 ~80% 证据）。4.0 让结构化只做一次。
> 跑这套流水线（P1→编译）的方法学精华见 [`../99-notes/写作流水线方法学/`](../99-notes/写作流水线方法学/)。
