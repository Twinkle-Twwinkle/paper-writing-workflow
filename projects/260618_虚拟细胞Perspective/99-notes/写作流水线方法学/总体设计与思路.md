# 00 · 总体设计与思路

## 要解决的问题
有 ~41 篇论文，每篇都有原文（`_cleaned.md`）和针对我们 context 提炼的 `_refined.md`。
**痛点**：让一个 LLM 一次读完 41 篇，context 直接爆。
**目标**：把它们整合成一篇**可审计**的 outline 骨架——每个论断对应真实论文引用 + 我们的引用编号 + 一句可回查的逐字原文；审计通过后能"编译"成完整的 1500–2000 字英文 Perspective。

## 6 阶段流水线
```
[地基 context]  ← 仅作"初始假设"，必须被真实证据修正（价值最大化的关键）
      │
P1 抽取   每篇派 1 个 subagent → 1 张结构化 JSON 证据卡（excerpt-centric）
      │        产物：_build/v4-cards/*.json + 合并 cards.json
      ▼
P2 矩阵   纯代码（零 agent）把 N 张卡按 claim 转置 → 证据矩阵 + scoreboard + 引用登记表
      │        产物：evidence-matrix.md / citation-registry.md
      ▼
P3 修正   1 个高推理 agent 看矩阵 + context → 裁决/锐化 thesis，点名要回击的反方 → thesis-revision.md
      │
      ▼
P4 成骨   每节一个 agent 写 outline 骨架：每句论断挂 [引用号] + 逐字原句（不是成文散文）
      │        产物：outline.md（可审计骨架）
      ▼
P5 审计   对抗式 agent 逐条查：引用真伪 / 原句真出自该篇 / 方向没用反 / 哪条 claim 证据薄
      │        产物：audit-report.md → 人工过
      ▼
[编译]   审计通过 → 展开成完整 perspective + 自动生成参考文献表
```

## 6 条核心设计原则（提炼技能时要保留的）

1. **每篇一个 subagent，主流程只持有"卡片"——context 永不爆。**
   不是"一个大脑读 41 篇"，而是 41 个小 agent 各读 1 篇、各吐一张 ~2KB JSON。这是"40 篇也能做"的物理根因。

2. **excerpt-centric JSON 是唯一结构化真相。**
   每条 verbatim 原句自带 `touchpoints:[{claimId,direction}]`（一句可同时支持一条 claim、反对另一条）。
   这把"摘录列表"和"claim→证据路由表"**合并成一个不会失同步的结构**。无 markdown 第二产物（要给人看就从 JSON 渲染）。

3. **能用确定性代码做的，绝不派 agent。**
   P2 的转置/计数/打旗标全是确定性操作 → 纯 Python（`build_matrix_v4.py`），零 token。改了某篇卡，重跑一行即重建矩阵。

4. **"可审计"= 每个论断物理绑定 [引用号] + 一句逐字原文。**
   审计不是去信 LLM，而是核对"这句原文真出自这篇吗（去 `_cleaned.md` 搜得到）、方向用对了吗"。
   逐字原句**带病照抄**（保留 OCR 残留），改了反而断了溯源链。

5. **context 是可被推翻的假设，不是教条。**
   P3 明确允许真实证据修正/增删/锐化初始 thesis——这让 perspective 价值最大化。技能里 context 是**运行时参数**，不焊死题目。

6. **增量搭建：写一个阶段 → 真跑一遍验货 → 看输出 → 改 → 下一阶段。**
   不闷头把 6 阶段全写完再跑。正是"跑一遍看 JSON"才暴露了 P1 冗余有损、claimId 碎裂、过度打标这些坑（见 `01`）。

## 两层引用体系（"真实引用 + 我们的编号"怎么落地）
- **第一层 = 我们的 41 篇**：`citationKey`（01–40，含 08a/08b）= 我们的引用编号 → `citation-registry.md` → 参考文献表主体。
- **第二层 = 论文内部引用**：每张卡的 `primarySources`（如 "scGPT → Cui H, Nat Methods 2024, ref#7"）。当 perspective 要引"某个一手事实"时，引的是这一层，而非二手转述。
- ⚠️ 41 篇里 33 篇书目是**推断**的（refined 自己 flag 了"从 DOI 前缀推"）→ 编译期必须核验（`citation-registry.md` 的 `meta?` 列已标）。

## 关键产物与角色
| 文件 | 角色 |
|---|---|
| `cards.json` / `v4-cards/*.json` | P1 产物，下游唯一数据源 |
| `evidence-matrix.md` | P3 主输入：按 claim 转置 + scoreboard + 逐字引文 + 涌现主题 |
| `citation-registry.md` | 参考文献种子 + 审计红线（书目存疑清单） |
| `build_matrix_v4.py` | P2 本体（确定性，含 canon()/逐字审计/过度打标探测） |
| `精练4.0-index.md` | 上游产卡技能（excerpt-centric JSON） |
