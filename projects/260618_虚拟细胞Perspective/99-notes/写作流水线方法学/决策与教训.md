> 提炼技能时，**坑和教训比成品脚本更值钱**。下面按"为什么这么决定"和"踩了什么坑、怎么修、留下什么规律"两块记。

## A. 关键决策（含理由）

| 决策 | 选择 | 理由 |
|---|---|---|
| 上游"精练"该不该直接吐 JSON？ | **是** | 原方案：精练产 markdown `_refined.md` → P1 用 41 个 agent 把 md 再解析回 JSON。发现这是**冗余往返**（~100 万 token）**且有损**（每 claim 只留 1 句，丢 ~80% 证据）。让精练直接产 JSON，P1 退化成近乎免费的拼接，结构化只做一次（在本来就深读那篇时）。 |

## B. 跑出来才暴露的 3 个坑（最重要）

### 坑① claimId 碎成 60 种 —— 枚举约束是承重的
- **现象**：P2 本该出 14 行 scoreboard，结果炸成 60+ 变体：`Axis A`/`axis A`/`modality`/`轴 A · 模态`/`蛋白质组`/`A` 全是同一个；`Claim 9`/`9`/`Claim 9 (data islands…)` 也散开。
- **根因**：旧 P1 的 schema 里 `claimId` 是 **`enum` 硬约束**，强制规范值。v4 改成"从 context 文件读标识符"——但 context 第4/5节是**散文+中文轴名+模态子表**，不是干净机读键，41 个 agent 各自翻译/编号 → 碎裂。
- **救本轮**：P2 里写 `canon()` 用规则把变体折回 14 规范键（**创可贴**，不重跑）。
- **根治（技能化时必做）**：context 文件带一份**规范 claim-id 清单**（干净机读键），精练技能"用且只用这些精确键"（或 workflow 在 schema 里注入 enum）。
- **🔑 规律**：**只要下游要做确定性聚合，claim/类别字段就必须用枚举或受控词表。** 自由文本必碎。这是把"通用、context 不焊死"和"机读可聚合"调和好的关键点。

### 坑② 过度打标 —— 用"支持论文数"而非"触点数"衡量
- **现象**：高召回下，个别论文给同一条 claim 堆 15–16 条 supports 触点（如 05/04 对 Claim 1；34 对 Claim 4 一度 22 条 challenges）。把 scoreboard 灌爆，让 claim 看起来被 N 方围攻、实则一个论点重复 N 遍。
- **缓解**：① 给精练加"**抑制过度打标**"规则（同一 (claim,direction) 不重复盖章，除非提供新证据——已写进 v4）。② P2 的 scoreboard 改用"**支持论文数**"（distinct papers），对单篇灌水免疫。
- **🔑 规律**：高召回抽取天然会在一篇论文的中心论点上过度产出；**衡量某 claim 有多硬，数"几篇不同论文支持"，不要数触点条数。**

### 坑③ 逐字审计的"假警报" —— 别把格式噪声当造假
- **现象**：逐字溯源审计读出 89%→93%，看着像保真度掉了。
- **真相**：深挖最低的论文（02、28）发现，所有"未命中"都是 **`<sup>2</sup>` 引用标签** 和 **粘在词后的裸引用号**（`Network13`、`analyses16`）被 agent 合理剥掉了。内容**逐字属实，零改写零编造**，真实保真 ≈ 100%。
- **修**：审计前对**源文件和原句都做归一化**——弯/直引号、长破折号、`<sup>/<sub>/<i>` 等格式标签（白名单，别误删 `<`/`>` 不等号）。
- **🔑 规律**：审计逐字保真前，**先归一化源端的格式垃圾**（标点字形 + HTML 标签 + 引用标记）；否则会把"格式差异"误判成"幻觉"。`build_matrix_v4.py` 的 `norm()` 已含这套归一化，可直接复用。

## C. Workflow 工具的坑（操作层）
- **`args` 必须传真 JSON 数组**，别传 JSON 字符串——否则脚本里 `args.map` 报 "is not a function"。脚本应防御：`Array.isArray(args)?args:JSON.parse(args)`。
- **`08` 是两篇**（`08a` Ståhl 2016 / `08b` MERFISH）→ 共 41 篇，citationKey 用 `^(\d+[a-z]?)` 提取。
- **并发上限** ~min(16, cores-2)，41 篇会分波跑；能用代码做的阶段（P2）别派 agent。
- 编辑持久化的 workflow 脚本后，用 `{scriptPath, resumeFromRunId}` 可缓存命中未改的 agent。

## D. 成本基线（供估算）
- P1 v4（41 篇重读全文 + 产卡）：**~274 万 token，~9 分钟**（并发约 10–14）。
- 旧 P1（读 refined 产有损卡，3 篇 demo）：~6.9 万/3 篇 ≈ 2.3 万/篇。
- P2：纯代码，~0 token。
